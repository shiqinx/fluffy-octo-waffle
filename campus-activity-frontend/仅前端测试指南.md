# 仅前端测试指南

## 🎯 纯前端测试方法

### 1. 切换到Mock模式
```bash
# 修改 .env.development
VITE_USE_MOCK=true
```

### 2. 纯前端测试项目
- ✅ 组件渲染测试
- ✅ 页面布局测试
- ✅ 用户交互测试
- ✅ 路由跳转测试
- ✅ 状态管理测试
- ✅ 样式响应式测试

### 3. Mock数据测试
- 使用项目内置的Mock数据
- 测试前端逻辑处理
- 验证数据展示格式
- 检查错误处理机制

### 4. 测试页面访问
```javascript
// 无需后端即可测试
http://localhost:3001/test-line-clamp.html
http://localhost:3001/test-activity-permissions.html
http://localhost:3001/verify-permissions.html
```

## 🔧 测试执行

### 重启前端服务
```bash
npm run dev
```

### 访问测试页面
直接在浏览器中打开测试页面，无需后端支持。

## 📊 测试范围

| 测试类型 | 是否需要后端 | 测试内容 |
|----------|-------------|----------|
| UI组件测试 | ❌ 不需要 | 组件渲染、交互 |
| 样式测试 | ❌ 不需要 | CSS、响应式 |
| 路由测试 | ❌ 不需要 | 页面跳转 |
| Mock数据测试 | ❌ 不需要 | 假数据处理 |
| API接口测试 | ✅ 需要 | 后端接口调用 |
| 认证测试 | ✅ 需要 | 登录权限验证 |
| 业务流程测试 | ✅ 需要 | 完整功能流程 |
